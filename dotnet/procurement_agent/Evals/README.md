# Agent365 Evaluation Framework

This directory contains the consolidated evaluation framework for Agent365, supporting both first-party (1P) and third-party (3P) evaluation methodologies.

## Directory Structure

```
Evals/
├── .gitignore                    # 🚫 CONSOLIDATED gitignore for entire framework
├── 1P Evals/                     # First-party LocalEvalRunner evaluations
│   ├── bin/                      # LocalEvalTool executable and configurations
│   ├── configs/                  # Evaluation configuration files
│   └── run-evaluation.ps1        # 1P evaluation script
├── 3P Evals/                     # Third-party direct GPT evaluations
│   ├── bin/                      # Configuration storage
│   └── run-evaluation.ps1        # 3P evaluation script
├── reports/                      # 📊 SHARED reports location
│   ├── .gitkeep                  # Preserves directory structure
│   ├── README.md                 # Reports documentation
│   └── *.json, *.txt             # Generated evaluation reports
├── scenarios/                    # 🎯 SHARED test scenarios
│   ├── agent365_basic_tests.json
│   ├── agent365_comprehensive_tests.json
│   ├── agent365_email_tests.json
│   ├── agent365_dataverse_mcp_tests.json
│   └── README.md                 # Scenarios documentation
├── ResultsViewer/                # 🌐 SHARED interactive results viewer
│   ├── Generate-ResultsViewer.ps1 # Results viewer generator
│   ├── template.html             # HTML template
│   ├── script.js                 # Interactive functionality
│   └── styles.css               # Styling
└── Generate-AgentConfig.ps1      # 🔧 SHARED agent configuration generator
```

## Evaluation Types

### 1P Evaluations (LocalEvalRunner)
- **Purpose**: Official evaluation using Microsoft's LocalEvalTool
- **Method**: Runs actual agent with mock plugins and advanced evaluators
- **Location**: `1P Evals/run-evaluation.ps1`
- **Features**: Function call tracking, detailed analysis, comprehensive reporting

### 3P Evaluations (Direct GPT)
- **Purpose**: Third-party evaluation using direct GPT simulation
- **Method**: Simulates agent behavior via Azure OpenAI GPT-4.1
- **Location**: `3P Evals/run-evaluation.ps1`
- **Features**: Rapid evaluation, similarity and quality assessment

## Shared Components

### 🎯 Test Scenarios (`scenarios/`)
- **Purpose**: Common test cases used by both evaluation types
- **Format**: JSON files with prompts, expected responses, and assertions
- **Benefits**: Consistent testing across evaluation methods

### 📊 Reports (`reports/`)
- **Purpose**: Unified location for all evaluation outputs
- **Formats**: JSON (machine-readable) and TXT (human-readable)
- **Access**: Used by both evaluation types and results viewer

### 🌐 Results Viewer (`ResultsViewer/`)
- **Purpose**: Interactive web-based results analysis
- **Features**: Visual charts, trend analysis, detailed comparisons
- **Usage**: Automatically generated by both evaluation types

### 🔧 Configuration Generator (`Generate-AgentConfig.ps1`)
- **Purpose**: Extracts agent configuration from source code
- **Output**: Agent configuration files for LocalEvalTool
- **Benefits**: Single source of truth, automatic plugin discovery

## Quick Start

### Run 1P Evaluation
```powershell
cd "1P Evals"
.\run-evaluation.ps1 -TestFile "agent365_basic_tests.json"
```

### Run 3P Evaluation
```powershell
cd "3P Evals"  
.\run-evaluation.ps1 -TestFile "agent365_comprehensive_tests.json"
```

### View Results
- **Interactive**: Open `ResultsViewer/index.html` in web browser
- **Reports**: Check `reports/` directory for detailed JSON/TXT files
